## 6. API-контракт `/api/order/`

**Эндпоинт:** `POST /api/order/` (и `POST /api/order`, слэш опционален).

**Content-Type варианты:**
- `multipart/form-data` — форма + до 6 файлов, суммарно ≤ 45 MB.
- `application/json` — без файлов, поля `OrderRequest`.

**Pydantic-модель (`api/routers/orders.py:35`):**
```python
class OrderRequest(BaseModel):
    workType: str = ""
    topic: str = ""
    subject: str = ""
    deadline: str = ""
    contact: str = ""   # единственное обязательное по смыслу поле
    comment: str = ""
```

**Константы:** `MAX_ORDER_FILES = 6`, `MAX_ORDER_TOTAL_SIZE = 45 * 1024 * 1024`.

**Rate-limit:** 3 заявки / IP / час → 429 `"Слишком много заявок. Попробуйте позже."`

**Успех:** JSON `{ok: true, id: <int>, attachments: [...]}`. После записи в SQLite + нотификация в каналы (Codex).

**Что НЕ менять во фронте без Codex:**
- Имена полей (`workType`, `topic`, `subject`, `deadline`, `contact`, `comment`).
- Путь (`/api/order/`).
- Лимиты файлов.
