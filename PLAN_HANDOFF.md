# PLAN_HANDOFF — пошаговая имплементация Dark Couture

> **Источник дизайна:** bundle из `/tmp/handoff/untitled/` (из Claude Design).
> **Авторитет:** `REAL_DATA.md` — факты, цены, контакты, API.
> **Ветка:** `design-handoff` от `main`. Все шаги коммитятся атомарно. Push и PR только по просьбе.

## Правила игры

1. **Один шаг = один сеанс агента = один коммит.** Если шаг не укладывается в 2–3 минуты — дроби.
2. **Никаких «заодно».** Рефакторинг, переименования и полировка — отдельные шаги.
3. **Всегда сверяться с `REAL_DATA.md`.** Выдуманные цены/контакты из мокапа → заменять на реальные.
4. **Тексты — только на русском.** Латынь/французский допустимы как декор (eyebrow, roman numerals), не как основной текст.
5. **Каждый шаг заканчивается `npm run build` в `astro-site/`.** Если сборка ломается — фикс перед коммитом, коммит не плодить.
6. **Не трогать `/doc/[slug]`, `api/`, `expo-app/`.** Работаем только в `astro-site/src/`.
7. **Фикс скоупа:** пока только главная `/`. Остальные страницы — отдельной серией шагов позже.

## Правила коммитов

Формат: `design(<section>): <imperative summary>`
Примеры:
- `design(tokens): add Dark Couture palette variables`
- `design(prelude): replace hero with editorial headline`
- `design(genres): add 8 genres list with real pricing`

Без `https://claude.ai/code/` в теле коммита, если пользователь не попросит.

---

## Критерий «готово» для каждого шага

1. ✅ `npm run build` проходит без ошибок и warnings.
2. ✅ Визуальная проверка `/` — новая секция на месте, соседние не сломаны.
3. ✅ Mobile-viewport 375px — без горизонтального скролла.
4. ✅ Тексты — русский, цифры и контакты — из `REAL_DATA.md`.
5. ✅ Один коммит, атомарный, с внятным сообщением.

## Чего НЕ делаем в handoff-ветке

- Рефакторинг других страниц (`/catalog`, `/order`, `/about` и т.д.) — отдельный план, позже.
- Смена API-контракта, новые эндпоинты — через Codex.
- Добавление зависимостей (если только сильно нужно — отдельным шагом с обоснованием).
- Трёхмерные эффекты, Three.js-сцены, WebGL — не в этой фазе.
