**Эталон:** `/tmp/reference/` — «Salon Académique · Dark Couture 2026», одностраничный лендинг.

**Цель:** перенести визуальный язык эталона на реальный контент; сохранить Astro-архитектуру и бесплатную библиотеку из 235 работ.

## Визуальные риски эталона (проверить перед переносом)

1. `h1 clamp(68px, 11vw, 184px)` — на 4K/ultrawide прыгает до 184px, переполняет hero. Добавить `max-width` по контейнеру.
2. `.prelude-head` — 4 колонки со статами; на 720–900 px риск схлопывания в невычитываемый ряд (media есть на 720, но 720–900 не покрыт — проверить).
3. `.calendar-months` — 12 кнопок-месяцев, на мобильном будет принудительный горизонтальный скролл.
4. `.archive-index` скрывается на `<900px` — значит `.archive-list` на мобильном должен иметь внутреннюю типографическую иерархию; проверить.
5. `.formula` — двухколоночный stage переходит в одну колонку на `<1100px`; «чек» окажется ниже слайдера — важно, чтобы сумма не уезжала под fold.
6. `body::before` (зерно) с `mix-blend-mode: overlay` на светлой теме даст грязь. **Наш сайт имеет light theme — зерно ограничить темой `[data-theme="dark"]`.**
7. Курсор эталона — не кастомный, наш тоже (решение совпадает).
8. Nav: `«Сейчас на связи 10:42»` — хардкод-плейсхолдер, JS обновляет через `#navTime`. Перенести `navTime` в Astro-скрипт.
9. Модалка заявки дублирует `/order` — на многостраничном сайте модалку делаем опциональной (FAB + прямой линк на `/order`).
10. Scroll-progress линия сверху — полезно, переносим.

## Фазы

Каждая фаза = один сеанс = один git-коммит на ветке `claude/setup-claude-project-VxGQV`. Сборка `npm run build` зелёная после каждой.

### Фаза 1 — Дизайн-фундамент (токены, базовые стили, шрифты)
**Файлы:** `astro-site/src/styles/design-tokens.css`, `layouts/Base.astro` (подключение шрифтов).
- Добавить токены эталона (`--ink`, `--ink-2…4`, `--cream`, `--cream-2/3`, `--mute`, `--gold-hi/lo/deep`, `--gold-line`, `--ease-silk`, `--wrap-wide/narrow`) рядом с существующими — не ломая light theme.
- Снизить радиусы для «editorial» блоков (новый набор `--r-edit-xs/sm/md`: 2/4/8). Существующие `--r 18 / --rm 16` — оставить для карточек каталога.
- Подключить Cormorant Garamond c italic-стилями 300–700 и swashes (`font-feature-settings: "swsh", "dlig"`).
- Глобальный `.serif / .sans / .mono / .eyebrow / .hairline / .hairline-gold` утилиты.
- Зерновая плёнка (`body::before` SVG turbulence) **только для `:root:not([data-theme="light"])`**.
- Scroll-progress `--p` линия в `Base.astro`.
**Готово, когда:** рендер текущих страниц не сломан; CSS-переменные эталона доступны глобально; build зелёный; DevTools показывает подключённые веса Cormorant.

### Фаза 2 — Навигация и футер в эталонном стиле
**Файлы:** `components/Nav.astro`, `components/Footer.astro`, `styles/nav.css`.
- Nav: бренд-mark «АС» (наш монограмм), римские номера глав у ссылок (`I Каталог`, `II Заказать`, `III Поделиться`, `IV О нас`, `V FAQ`), справа `nav-hours` + CTA «Заказать».
- Burger-drawer (≤1180 px) с теми же ссылками + римскими цифрами.
- Hairline-бордер nav, `nav.scrolled` с `backdrop-filter`.
- `nav-time` — живой часовой индикатор через `<script is:inline>` с `setInterval` (только локальное время по МСК).
- Footer (editorial): 4 колонки (Мастерская / Каталог / Документы / Контакты), copyright `© MMXX—MMXXVI · Академический Салон`, приписка «Сделано тихо, для тех, кому важен текст».
**Готово, когда:** все страницы имеют новый nav/footer, active-state по текущей странице (не по скроллу — у нас мультистраничник), drawer открывается/закрывается, время тикает.

### Фаза 3 — Главная: hero + манифест + метод + гарантии
**Файлы:** `pages/index.astro`.
- Новый hero (`prelude`): eyebrow «Мастерская · с 2020», H1 в три строки с italic-акцентами, 3 стата вместо 4 (`В каталоге — 235 работ / С 2020 года / Ответ за 15 мин`), две CTA: «Смотреть каталог» (primary) и «Заказать работу» (ghost), scroll-cue.
- Манифест (`Caput Primum`): drop-cap на первой букве, 3 абзаца со стилем «пишем медленно», блок `tenets` (3 принципа).
- Метод (`Caput Secundum · Как работает библиотека`): **адаптируем под реальность** — 4 шага вместо 5: I. Ищите в каталоге, II. Скачивайте, III. Не нашли — закажите, IV. Сопровождение до защиты. Табы + панель с деталями.
- Гарантии: 4 `§`-карточки (оригинальность, сроки, правки, конфиденциальность) — без фразы «за 15 лет такого не случалось».
- Секция «Отзывы» — оставляем текущую бегущую стену (`reviews/1-10.webp`), только меняем обёртку под editorial-стиль: eyebrow «Caput Tertium · Свидетельства», серифный заголовок.
- Убираем секцию `order-prices` в её текущем виде — заменяется в фазе 5.
**Готово, когда:** главная визуально соответствует эталону в секциях 1-4; «15 лет» нигде нет; две CTA работают; mobile ≤720 px — без горизонтального скролла.

## Порядок и зависимости

```
Фаза 1 ─► Фаза 2 ─┬─► Фаза 3 ─┬─► Фаза 7
                  ├─► Фаза 4 ─┤
                  ├─► Фаза 5 ─┤
                  └─► Фаза 6 ─┘
```

Фазы 3–6 можно делать параллельно разными сессиями после 1+2. Фаза 7 — финальная сборка.

Рекомендуемая очерёдность для одного разработчика: **1 → 2 → 3 → 5 → 4 → 6 → 7**. Обоснование: после фазы 3 главная уже продающая; фаза 5 критична для конверсии; каталог (4) — самая объёмная, делаем когда остальное стоит.

## Критерии «не сломать»

- Не трогать `api/`, `stats_api.py`, `expo-app/` — зона Codex.
- Не деплоить руками через `scp` — только `git push main` → GitHub Actions.
- Сохранить все существующие URL (`/catalog`, `/doc/*`, `/order`, `/about`, `/contribute`, `/admin`).
- Не удалять `public/reviews/*.webp`, `public/files/*`.
- Light theme продолжает работать (тестировать переключатель после каждой фазы).
- `D.length` остаётся единым источником правды для счётчика работ.
